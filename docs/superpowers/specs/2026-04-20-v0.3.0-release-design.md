# Design Spec: v0.3.0 Release

## Goal
Perform a proper release of the CoconutClaw runtime by synchronizing the version in `Cargo.toml` with a new git tag `v0.3.0`.

## Context
- Current `Cargo.toml` version: `0.2.0`
- Latest git tag: `v0.2.3`
- Desired release version: `0.3.0`

## Implementation Plan

### 1. Quality Gates (Pre-release)
- Run `make lint` and `make test` to ensure the codebase is in a release-ready state.

### 2. Version Update & Changelog
- Modify `Cargo.toml` (root):
  - Change `[workspace.package].version` from `"0.2.0"` to `"0.3.0"`.
- Initialize `CHANGELOG.md`:
  - Create the file with initial entries for v0.3.0 (e.g., "Implement agent-driven scheduled tasks", "Improve Slack integration").

### 3. Synchronization
- Run `cargo check` to update `Cargo.lock`.

### 4. Commit
- Commit changes with message: `chore: bump version to 0.3.0 and initialize changelog`

### 5. Tagging
- Create annotated tag: `git tag -a v0.3.0 -m "Release v0.3.0"`

### 6. Verification & Push
- Build release binary locally: `make release`
- Push commit and tags to origin: `git push origin main && git push origin v0.3.0`

## Success Criteria
- Root `Cargo.toml` shows version `0.3.0`.
- `CHANGELOG.md` exists and contains v0.3.0 entries.
- Git tag `v0.3.0` exists as an annotated tag.
- `make release` completes successfully.
- Changes are pushed to the remote repository, triggering the release workflow.
